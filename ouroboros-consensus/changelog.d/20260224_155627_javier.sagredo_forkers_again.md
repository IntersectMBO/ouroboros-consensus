### Breaking

- ChainDB API: remove `getReadOnlyForkerAtPoint` in favor of
  `allocInRegistryReadOnlyForkerAtPoint` and `withReadOnlyForkerAtPoint`.
- LedgerDB API: remove `getForkerAtTarget` in favor of `unsafeGetForkerAtTarget`
  which no longer expects a `ResourceRegistry`.
- LedgerDB initialization now runs in `WithTempRegistry st m`.
- `ValidateArgs`: remove `resourceReg` for a continuation `onSuccess` that will
  be run if the candidate is fully valid.
- The continuation in `localStateQueryServer` now returns a `ResourceKey` as it
  should allocate the resource in a `ResourceRegistry`.
- Mempool: `LedgerInterface` no longer expects a `ResourceRegistry`. Instead a
  registry will be used by the action in `mldViewGetForker`.

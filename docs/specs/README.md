# Consensus specifications

Literate [Quint](https://quint-lang.org/) specifications for Ouroboros Consensus.
Each `.md` file is a literate spec: prose interleaved with Quint code blocks.
The code blocks are extracted into `.qnt` files for typechecking, simulation, and model checking.

## Prerequisites

Enter the nix shell (provides `lmt` and `quint`):

    nix develop

## Quick start

Extract, typecheck, and test all specs:

    make

## Individual commands

Extract `.qnt` files from the literate sources:

    make extract

Typecheck:

    make check

Run tests:

    make test

## Simulation

Run the Praos model (3 nodes, k=2, 20 slots) for 20 steps:

    make sim

This runs `init`, then repeatedly picks a random action (block production, chain delivery, or slot advance).
The output shows only `chains` (the interesting state); other variables are hidden.
Add `--seed=0x...` to reproduce a specific trace:

    quint run --main PraosRun --max-steps=20 --seed=0xabc praos.qnt --hide PraosRun::Praos::messages PraosRun::Praos::hashCounter PraosRun::Praos::slot

## Interactive exploration

Start a Quint REPL with the Praos model loaded:

    make extract
    quint -r praos.qnt::PraosRun

## Files

- `praos.md` — top-level Praos network model (literate source)
- `Makefile` — build automation
- `*.qnt` — extracted Quint files (gitignored, regenerated by `make extract`)

### Breaking

- Consolidate the table kind machinery into a single `Ouroboros.Consensus.Ledger.Tables` module and reshape `LedgerTables`. The former `MapKind` (a 2-arg key/value type-level function) is replaced by `TableKind` (a 1-arg type indexed by `blk`), and the concrete map kinds are renamed and re-indexed accordingly.
  - New module `Ouroboros.Consensus.Ledger.Tables` subsumes `Ouroboros.Consensus.Ledger.Tables`, `Ouroboros.Consensus.Ledger.Tables.Basics`, `Ouroboros.Consensus.Ledger.Tables.Combinators`, `Ouroboros.Consensus.Ledger.Tables.Kinds`, `Ouroboros.Consensus.Ledger.Tables.MapKind`, and `Ouroboros.Consensus.Ledger.Tables.Utils`.
  - Renamed: `EmptyMK` → `NoTables`, `KeysMK` → `Keys`, `ValuesMK` → `Values`, `DiffMK` → `Diffs`. The new types take a single `blk` parameter directly (e.g. `Values blk` instead of `ValuesMK (TxIn blk) (TxOut blk)`).
  - Renamed: `CanMapKeysMK` → `CanMapKeys`, `mapKeysMK` → `mapKeys`.
  - New typeclass `BimapTables` replaces the prior `bimapLedgerTables` machinery; new typeclass `EmptyTable` replaces `ZeroableMK`.
  - `Ouroboros.Consensus.Ledger.Tables.Diff` now exports `empty`, `map`, and `mapKeys` helpers.
  - `Ouroboros.Consensus.Util` exports a new constant bifunctor `K2`.
